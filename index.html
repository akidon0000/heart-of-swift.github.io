Heart of Swift

はじめに

Swift は 値型 を中心とした特徴的なプログラミング言語です。他のプログラミング言語の経験がある人でも、 Swift がどのようなコンセプトの言語なのかを理解しないままにコードを書くとその違いに戸惑い、 Swift らしくない（ Swifty でない）コードを書いてしまいがちです。そこで "Heart of Swift" と題して、筆者が Swift を学ぶ上で最も重要だと思う言語の Heart をオンラインブックの形でまとめました。

Heart という単語には、「心臓」の他に「心」や「中心」などの意味があります。

heart

名詞
  1. 心臓
  2. 心, 気持ち
  3. [the 〜] 中心, 真ん中

"Heart of Swift" というタイトルには、 Swift の根幹をなす概念（中心）や、 Swift という言語がプログラマにどのようなコードを書くことを期待しているか（心）などの意味を込めています。本書が目指すのは、 Swift の　Heart （中心）となる概念について説明することを通して、 Swift の Heart （心）の形を描き出すことです。

では、 Swift の Heart （中心）となる概念とは何かというと、それが次の二つです。

  • Value Semantics
  • Protocol-oriented Programming

Value Semantics と Protocol-oriented Programming については、 WWDC 2015 のセッションの中で詳しく説明されています。

  • "Building Better Apps with Value Types in Swift"
    Doug Gregor & Bill Dudney, Session 414, WWDC 2015
  • "Protocol-Oriented Programming in Swift"
    Dave Abrahams, Session 408, WWDC 2015

これらのセッションの発表者の内、赤字の二人は Swift Core Team のメンバーです。 Swift の言語仕様の変更や追加は Swift Evolution で話し合われるのですが、その議論を統括して最終的な意思決定を下すのが Swift Core Team です。 Swift の生みの親である Chris Lattner さんも Core Team のメンバーの一人であり、まさに Core Team は Swift の中枢と言えます。上記の二つのセッションは、 Swift とはどういうコンセプトの言語なのかを Core Team のメンバーが自らの口で語った、数ある Swift 関連のセッションの中でも特に重要な二つだと筆者は考えています。

しかし、 2015 年当時から Swift は大きく進化しました。当時の Swift のバージョンは 1.2 です。当然ですが、それらのセッションには Swift の最新事情は反映されていません。たとえば、 Swift 5.1 で Opaque Result Type （ SwiftUI などで多用される some View のような some の付いた型）が導入されました。 Opaque Result Type は Value Semantics と Protocol-oriented Programming の延長上にある存在ですが、 2015 年当時に Opaque Result Typeと Value Semantics や Protocol-oriented Programming の関係を論じることはできません。

そこで本書は、 Value Semantics と Protocol-oriented Programming を軸に Swift の最新事情を考慮して話を再構築し、 Swift という言語が持つコンセプトの全体像を示そうと試みました。本書が、 Swift という言語を理解する手助けになることを願っています。
